# Phase 1.2.1 — Database Initialization Fix

## Root cause

Garry's Mod's `sql.LastError()` can retain an error generated by another addon.
The first database abstraction treated any non-empty `sql.LastError()` value as
a failure even when the current `sql.Query()` succeeded.

That caused Convergence initialization to stop before saving schema metadata and
setting the ready state, while ordinary planet reads and writes still worked.

## Fix

- Treat `sql.Query(...) == false` as the authoritative SQLite failure signal.
- Split startup into explicit connection, metadata, migration, and bootstrap stages.
- Add stage-specific diagnostic output.
- Use Garry's Mod `sql.Begin()` and `sql.Commit()` helpers.
- Preserve all existing planet and stability-history data.

## Expected diagnostics

```text
Addon version:      0.1.2
Target schema:      1
Database adapter:   sqlite
Connection:         PASS
Metadata:           PASS
Migrations:         PASS
Planet bootstrap:   PASS
Database ready:     PASS
Installed schema:   1
```
